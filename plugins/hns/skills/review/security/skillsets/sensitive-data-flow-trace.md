# Sensitive Data Flow Trace

## Procedure
1. 민감 데이터 식별: PII (이름, 이메일, 전화번호), 결제 정보 (카드번호, CVV), 인증 토큰, 비밀번호
2. 데이터 흐름 추적 (각 단계별):
   - **수집**: 어디서 입력받는가? 최소 수집 원칙 준수?
   - **전송**: TLS 적용? 서비스 간 통신에서 평문 전달?
   - **처리**: 메모리 내 노출 범위? 불필요한 복사/변환?
   - **저장**: 암호화/해싱 적용? 접근 권한 제한?
   - **로깅**: 민감 데이터 마스킹? 로그 레벨별 노출 범위?
   - **삭제**: 보존 기간 정의? 삭제 메커니즘?
3. 서비스 경계 넘어가는 민감 데이터 확인 (Kafka 메시지, API 응답에 불필요한 필드 포함 여부)

## Pass Criteria
- 민감 데이터가 불필요하게 서비스 경계를 넘지 않음
- 저장/전송 시 적절한 암호화 적용
- 로그에 민감 데이터 마스킹 확인
