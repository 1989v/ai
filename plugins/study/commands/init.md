# Study — Init (학습 계획서 생성)

study/temp.md에 기록된 스터디 주제를 읽고, 구조화된 학습 계획서를 `study/docs/` 하위에 생성한다.

## 트리거

- `/study:init` — 학습 계획서가 아직 없는 주제 목록을 보여주고 선택
- `/study:init 1` — 1번 주제의 학습 계획서를 바로 생성
- `/study:init all` — 미생성 주제 전체 일괄 처리
- 자연어: "스터디 정리해줘", "학습 계획 만들어줘", "주제 초기화", "스터디 시작"

## 실행 단계

### 1. 현황 파악

1. `study/temp.md` 파일을 읽는다 (Read 도구)
2. `study/docs/` 디렉토리를 스캔하여 이미 생성된 학습 계획서 목록을 확인한다 (Glob 도구)
3. temp.md에서 아직 학습 계획서가 없는 주제를 식별한다

### 2. 사용자 프로필 로딩

메모리에서 사용자 정보를 로드하여 학습 수준을 파악한다:
- 현재 역할/직급
- 기술 스택 경험
- 프로젝트 도메인

이 정보를 학습 계획서의 난이도 설정과 깊이 조절에 활용한다.

### 3. 주제 선택

- **번호가 주어진 경우**: 해당 번호의 주제를 바로 처리
- **번호가 없는 경우**: 미생성 주제 목록을 표 형태로 출력하고 사용자에게 선택을 요청

출력 예시:
```
| # | 주제 | 계획서 상태 |
|---|------|-----------|
| 1 | AWS 네트워크 인프라 (VPC, NAT, ALB/NLB) | 미생성 |
| 2 | Kafka 내부 동작 원리 | 미생성 |
```

### 4. 학습 계획서 생성

선택된 주제의 temp.md 내용을 기반으로 학습 계획서를 작성한다.

**파일 경로**: `study/docs/{번호}-{slug}.md`
- slug는 주제 핵심 키워드의 영문 kebab-case (예: `1-aws-network.md`, `2-kafka-internals.md`)

**학습 계획서 템플릿**:

```markdown
---
id: {번호}
title: {한글 제목}
status: draft
created: {YYYY-MM-DD}
updated: {YYYY-MM-DD}
tags: [태그1, 태그2, 태그3]
difficulty: {beginner | intermediate | advanced}
estimated-hours: {예상 학습 시간}
codebase-relevant: {true | false}
---

# {한글 제목}

## 1. 개요
<!-- 이 주제가 무엇이고 왜 학습이 필요한지 1-2문단 -->

## 2. 학습 목표
<!-- 학습 완료 후 달성해야 할 구체적 목표 (bullet) -->
<!-- 예: "VPC 서브넷 설계를 직접 할 수 있다" -->

## 3. 선수 지식
<!-- 이 주제를 학습하기 전 알아야 할 개념 -->

## 4. 학습 로드맵
### Phase 1: 기본 개념
<!-- 핵심 용어, 기초 이론, 동작 원리 -->

### Phase 2: 심화
<!-- 내부 구조, 성능 최적화, 트레이드오프 -->

### Phase 3: 실전 적용
<!-- 현재 프로젝트/코드베이스와 연관된 실습 포인트 -->

### Phase 4: 면접 대비
<!-- 자주 나오는 질문 유형, 꼬리 질문 패턴 -->

## 5. 코드베이스 연관성
<!-- 현재 msa 프로젝트에서 이 주제와 관련된 모듈/설정/코드 위치 개략 -->
<!-- codebase-relevant: false이면 "직접 연관 코드 없음 — 인프라/이론 주제" -->

## 6. 참고 자료
<!-- 공식 문서, 추천 글, 영상 등 (있으면) -->

## 7. 미결 사항
<!-- 브레인스토밍에서 논의가 필요한 항목 -->
<!-- 학습 범위, 깊이, 우선순위 관련 질문 -->

## 8. 원본 메모
<!-- temp.md에서 가져온 원문 그대로 보존 -->
```

**작성 원칙**:
- temp.md의 원본 내용을 `8. 원본 메모` 섹션에 그대로 보존
- 섹션 1~5는 원본 내용을 기반으로 AI가 구조화하여 작성
- `difficulty`는 사용자 프로필과 주제 내용을 종합하여 판단
- `codebase-relevant`는 현재 msa 프로젝트와의 관련성 여부 (Grep/Glob으로 빠르게 체크)
- 학습 로드맵은 구체적 소주제를 bullet으로 나열 (아직 본문 작성은 아님)
- 과도한 추측이나 없는 내용 창작 금지 — 원본에 근거한 정리만 수행

### 5. temp.md 현황 표 업데이트

학습 계획서 생성 후, `study/temp.md`의 **학습 현황** 표에 해당 주제의 행을 추가/업데이트한다 (Edit 도구).

```markdown
## 학습 현황

| # | 주제 | 계획서 | 상태 |
|---|------|-------|------|
| {번호} | {한글 제목} | [{번호}-{slug}.md](docs/{번호}-{slug}.md) | draft |
```

- 이미 해당 번호의 행이 있으면 업데이트
- 없으면 번호 순서에 맞게 행 추가
- 학습 현황 표 자체가 없으면 주제 목록 아래에 생성

### 6. 결과 보고

학습 계획서 생성 후 사용자에게:
- 생성된 파일 경로
- 학습 목표 요약 (3줄 이내)
- 예상 난이도 및 학습 시간
- 코드베이스 연관성 여부
- 다음 단계 안내: `/study:bs {번호}` 로 학습 방향 브레인스토밍 가능

### 7. 다수 주제 일괄 처리

`/study:init all` 호출 시:
- 학습 계획서 미생성인 모든 주제에 대해 순차적으로 생성
- 각 생성 후 학습 현황 표에 행 추가
- 완료 후 전체 현황 표 출력

## 에러 처리

- `study/temp.md`가 없는 경우: 파일 생성 안내
- `study/docs/` 디렉토리가 없는 경우: 자동 생성
- 이미 해당 번호의 학습 계획서가 존재하는 경우: 덮어쓰기 여부 확인
- 존재하지 않는 번호 지정 시: 사용 가능한 번호 목록 출력
