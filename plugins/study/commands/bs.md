# Study — Brainstorm (학습 방향 브레인스토밍)

이미 생성된 학습 계획서를 사용자와 대화형으로 검토하여 학습 방향을 가다듬는다.

## 트리거

- `/study:bs` — draft/refined 상태인 학습 계획서 목록을 보여주고 선택
- `/study:bs 1` — 1번 계획서 바로 브레인스토밍
- 자연어: "스터디 방향 잡자", "학습 범위 조정하자"

## 실행 단계

### 1. 계획서 선택

- `study/docs/*/plan.md` 스캔 → frontmatter status 확인
- 번호 주어지면 `study/docs/{N}-{slug}/plan.md` 바로 열기

### 2. 사용자 프로필 확인

메모리에서 사용자 정보 로드.

**첫 브레인스토밍 시 반드시 확인**: 해당 주제 현재 수준 (기초/중급/고급)

### 3. 계획서 분석 및 대화 시작

1. 현재 학습 계획 요약 (3-5줄)
2. 미결 사항 (섹션 7)
3. 첫 번째 질문 (학습 방향에 가장 큰 영향)

### 4. 핑퐁 브레인스토밍

**원칙**:
- 한 번에 한 가지 질문만
- 가능하면 선택지 제공 (A/B/C)
- 답변 받을 때마다 즉시 `plan.md` 업데이트
- 사용자가 "됐어", "완료" → 종료

**논의 흐름**:

**Phase A: 범위 정의**
- 깊이 (개념/실무/면접/전체)
- 시간 제약
- Phase 우선순위

**Phase B: 깊이 조절**
- 이미 아는 영역 (스킵 가능)
- 꼭 포함할 심화 주제
- 코드베이스 탐색 범위

**Phase C: 면접 방향** (해당 시)
- 목표 포지션/회사 특성
- 예상 면접 유형
- 꼬리 질문 깊이

**Phase D: 학습 전략**
- 출력 형태 (노트/Q&A 카드/치트시트)
- 실습 포함 여부

### 5. 계획서 업데이트

- 답변 즉시 해당 섹션에 반영
- frontmatter의 `updated`, `difficulty`, `estimated-hours`, `learner-level` 갱신
- 로드맵 Phase 의 소주제 조정
- 충분히 가다듬어지면: `status: draft → refined`
- 학습 시작 준비 완료 선언 시: `status: refined → ready`
- **상태 변경 시 `study/temp.md` 학습 현황 표도 동기화**

### 6. 종료

- 이번 세션 결정사항 요약
- 현재 계획서 상태
- 조정된 시간 / 난이도
- 다음 단계:
  - `refined`: 추가 `/study:bs {N}` 가능
  - `ready`: `/study:exec {N}` 로 프리뷰 생성 가능

## 상태 흐름

```
draft → refined → ready → in-progress (exec/start) → completed
```

## 대화 스타일

- 멘토 톤 (시니어 동료)
- 사용자 수준 존중 + 한 단계 위 도전 권유
- "이건 이미 아시죠?" 보다 "혹시 이 부분 복습 필요할까요?"
- 면접 관점 "이걸 물어보면 어떻게 답하실 건가요?" 사전 점검
- 더 효율적인 학습 경로 적극 제안

## 에러 처리

- `study/docs/{N}-{slug}/plan.md` 없음: `/study:init {N}` 안내
- `study/docs/` 비어있음: `/study:init` 먼저 실행 안내
